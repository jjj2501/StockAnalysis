# 微信公众号自动写作与排版指南 (Skill)

本技能/工作流旨在规范 AI 辅助创作微信公众号文章时的**内容结构、阅读体验**与**Markdown 格式兼容性**。公众号文章的受众大多在手机端碎片化阅读，因此对文章的通俗易懂性、排版留白和视觉引导要求极高。

## 适用场景
- 撰写技术技术分享、项目更新和复盘总结
- 推广新功能发布（如本次 AlphaPulse 训练工具上线）
- 输出行业观察和深度观点长文

---

## 💡 一、黄金文章结构模板

一篇高完播率、易于转发的公众号推文，应遵循以下结构：

1. **吸睛标题 (Headline)**：< 25 字，必须带利益点或悬念，可叠加适用 Emoji。例如：*《终于搞定！这套量化选股架构，帮我省下了 90% 的盯盘时间》*
2. **引言/痛点引入 (Hook)**：抛出读者常见的困境，建立共鸣。字数控制在 100-200 字。
3. **核心亮点摘要 (TL;DR)**：如果文章较长，在开头提供 3 句以内的一句话总结，直接抛出核心结论。
4. **正文拆解 (Body Paragraphs)**：
   - 采用递进式或并列式小标题。
   - 每段字数**不宜超过 3-4 行（手机屏幕阅读宽度）**，强迫自己分段，多用回车留白。
5. **图文并茂 (Visuals)**：纯文字极易让人疲劳，适量插入系统界面截图、流程图（甚至表情包）。
6. **结语与互动 (Call to Action - CTA)**：
   - 总结全文。
   - 抛出一个开放式问题引导留言。
   - 提示读者“点赞、在看、转发”或引导点击“阅读原文”/扫码关注。

## 📝 二、适配公众号的 Markdown 排版规范

微信自带的编辑器对 Markdown 兼容性较弱，通常我们需要利用**外部转换工具（如 Md2All、墨滴、微信公众号排版助手插件等）**。因此在生成 MD 源文件时，必须注意以下几点：

### 1. 标题层级管理
- 最多使用 2 层标题（`##` 模块标题 和 `###` 段落小标题）。
- 不要在标题内使用复杂的 Markdown 语法或超长句子。

### 2. 重点突出 (强调视觉中心)
在全篇平淡的文字中，必须手动打造视觉落脚点（通常用**加粗**或特定颜色标注）。
> **最佳实践**：凡是核心结论、步骤说明、关键数字，均使用 `**加粗文本**`。

### 3. 内容的分块与引用
使用引用块 `>` 来放置：
- 核心金句 / 名人名言
- 注意事项 (Tip/Warning)
- 总结语

> 💡 **提示**：引用块在经过公众号排版工具转换后，通常会生成带有左侧高亮边框和极佳浅色背景的优雅版块，非常吸引眼球。

### 4. 避免使用复杂的表格
如果不是必须，尽量不要在公众号使用超过 3 列的表格。在手机屏幕上，表格极易因为过宽而导致需要横向滑动，阅读体验极差。**替代方案：使用加粗列表。**

**❌ 错误示范：**
| 功能 | 以前 | 现在 |
|-----|------|------|
| GPU检测 | 无 | 全自动设别 |

**✅ 正确示范：**
- **GPU 检测：** 从以前的[无] 升级为目前的 [全自动识别并加载]。

## 🤖 三、一键生成推文的 Prompt 模板

当你需要让 AI 根据你的代码更新或需求，直接输出一篇即发公众号草稿时，请发送以下指令：

```text
角色：你是一名深谙微信生态的资深新媒体主编，同时精通技术领域。

任务：请根据以下【素材】，为我写一篇要在微信公众号发布的文章。

排版要求：
1. 请提供 5 个候选标题（包含夸张版、痛点型、技术干货型）。
2. 文章必须要包含：痛点引入、功能拆解（并列 3 点最佳）、简单总结、引导关注。
3. 段落必须极度切碎！一句话一段，每段不要超过 30 个字。
4. 核心术语、震撼的数据必须使用 **加粗** 处理。
5. 在需要放截图的地方，用 [此处插入: xxx的截图] 占位提示我。
6. 整体语气轻松、有网感，像是一个大佬在跟学弟分享经验。

【素材】：（在这里贴上你最近完成的更新日志，例如修复了 GPU 识别，加了训练页面等）
```

## 🛠️ 四、发文前 Checklist
- [ ] 标题是否能在 1 秒内抓人眼球？
- [ ] 是否所有的段落都在手机屏幕的一屏以内？（也就是段落没有过长变“黑砖头”）
- [ ] 代码块是否已经经过高亮工具处理，未导致横向溢出？
- [ ] 是否在文末放置了与读者的互动话题？
- [ ] 是否添加了封面封面图和合适的摘要？
